Краткое описание

  • Есть эксперт, который может помочь человеку выбрать наиболее подходящий объект.

Краткое описание

  • Есть много Объектов, среди которых нужно выбрать наиболее подходящий

Краткое описание

  • У Объектов есть Атрибуты:

Краткое описание

  • Есть пользователи экспертной системы

Краткое описание

  • У пользователей есть параметры

Краткое описание

  • Эксперт обладает великим знанием преобразования параметров пользователя в атрибуты объектов.

Что делать? Внести в систему:

  • Объекты с атрибутами
  • Параметры
  • Вопросы с ответами (для выяснения параметров пользователя)
  • Правила получения атрибутов из правил (великое знание)
  • * Еще можно задать правила получения новых параметров из выясненных

Как вносить:

  • Нажать на большой плюс

Как вносить:

  • Заполнить название и описание системы, нажать "САБМИТ!"

Как вносить:

  • На вкладке "Атрибуты" заполнить атрибуты, "Сабмит!"

Как вносить:

  • На вкладке "Объекты" добавить объекты и атрибуты, "Сабмит!"

Как вносить:

  • На вкладке "Параметры" добавить параметры, "Сабмит!"

Как вносить:

  • Добавить параметрам вопросы (2 типа), "Сабмит!"

Как вносить:

  • Добавить ответы, каждый ответ задает значение параметра, "Сабмит!"

Как вносить:

  • Добавить правила, "Сабмит!"

Требования Терехова

  • P >=20 Количество параметров, извлекаемых от пользователя, должно быть более 19
  • A >=10 Количество атрибутов должно быть более 9
  • P > 2*A Количество исходных данных должно быть больше количества необходимых атрибутов для принятия решения больше как минимум в два раза
  • Множество A может только частично (не более чем на 30%) входить в множество P.
  • Количество вопросов должно быть не меньше количества параметров P.
